GetDVTable

The following routes are available for this service:
GET /GetDVTable
GetDVTable Request Parameters:
Name Parameter Data Type Required Description
userName query string Yes AQUARIUS user name.
password query string Yes AQUARIUS password.
LocationIdentifier query string No AQUARIUS location identifier.
Parameter query string No AQUARIUS parameter.
QueryFrom query ISO 8601 time string No AQUARIUS start datetime.
QueryTo query ISO 8601 time string No AQUARIUS end datetime.
GetDVTable Response:

The GetDVTable response, when successful, returns an RDB file (MIME content type “text/plain”). The RDB file begins with a sequence of RDB comment lines, forming a “header block”, which contain additional data relevant to all daily values in the subsequent table.

If unsuccessful, aq2rdb will attempt to respond with an error message included in a single RDB comment line.

An example header block, is:

# //UNITED STATES GEOLOGICAL SURVEY http://water.usgs.gov/
# //NATIONAL WATER INFORMATION SYSTEM http://water.usgs.gov/data.html
# //DATA ARE PROVISIONAL AND SUBJECT TO CHANGE UNTIL PUBLISHED BY USGS
# //RETRIEVED: 2015-11-16 20:54:43Z
# //FILE TYPE="NWIS-I DAILY-VALUES" EDITABLE=NO
# //STATION AGENCY="USGS" NUMBER="01010000 " TIME_ZONE="EST" DST_FLAG=Y
# //STATION NAME="St. John River at Ninemile Bridge, Maine"
# //RANGE START="19960101" END="19970101"

The RETRIEVED field contains the time that the aq2rdb query was made. Please note that unlike nwts2rdb, the RETRIEVED time is always expressed in UTC (“Z” is the U.S. military time zone abbreviation for the ï¿½00:00 time offset), as it would not be accurate for aq2rdb to attempt to detect the local time zone of the Web service client requesting the daily values file.

The remaining header block fields are a subset of those output by the NWIS nwts2rdb program.

The columns included in the RDB table are:

Name Parameter Data Type Required Description
DATE RDB column 8D Yes Daily value date.
TIME RDB column 6S No Daily value time.
VALUE RDB column 16N No Daily value.
REMARK RDB column 1S No Remark code.
FLAGS RDB column 32S No Flags.
TYPE RDB column 1S No Type.
QA RDB column 1S No QA.
